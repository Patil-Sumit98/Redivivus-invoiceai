Hardening InvoiceAI: Wave 1-4 Security & Technical Debt Remediation

This commit applies all 22 bug fixes and hardening tasks outlined in the execution plan to achieve production readiness.

Key Changes:

Wave 1 (Critical Security):
- VUL-01: Removed raw Azure Blob URLs from DB. `blob_storage.py` now stores `blob_name` and generates time-limited SAS URLs dynamically. Updated `invoices.py` and frontend to utilize the SAS URLs.
- VUL-02: Hardened SSRF protection. Webhook URL validation (`webhook_service.py`) now resolves DNS to block private, link-local, loopback, and Azure metadata IPs, preventing DNS rebinding attacks.
- VUL-03: Resolved OOM vulnerability. Background tasks no longer receive massive file byte arrays; they stream data from blob storage.

Wave 2 (High Severity):
- BUG-04: Fixed GST tax split logic in `invoice_mapper.py` to correctly extract CGST, SGST, and IGST values directly from Azure fields before falling back to manual derivation.
- BUG-05: Increased max invoice date window from 180 to 1095 days in `gst_rules.py` for audit/reconciliation workflows.
- BUG-06: Implemented secure refresh tokens. Shortened JWT access token to 60m and added a 7-day httpOnly refresh token cookie, updating both backend auth routes and frontend Axios interceptors.
- BUG-07: Unified processing pipeline in `processing_pipeline.py`. Reprocess tasks now successfully run the free QR detection step before resorting to expensive Azure AI OCR.
- BUG-08: Introduced Redis-backed sliding window rate limiter in `rate_limit.py` with a fail-open in-memory fallback.
- BUG-09: Updated `invoices.py` list endpoint to search within `data_json` fields (vendor, GSTIN, invoice number).
- BUG-10: Created a singleton `BlobServiceClient` with `@lru_cache` to eliminate connection pool exhaustion.

Wave 3 & 4 (Medium & Low Severity):
- BUG-11: Added `server_default=func.now()` to the `updated_at` column in the `Invoice` schema.
- BUG-12: Enforced strong passwords via Pydantic validator and added an interactive frontend password strength meter.
- BUG-13: Removed hardcoded `default-secret` fallback for webhooks and implemented secure 32-byte hex generation.
- BUG-14: Refactored data exports to stream CSVs in batches and added a 10K record cap for XLSX to prevent OOM errors.
- BUG-15: Enhanced `SecurityHeadersMiddleware` with HSTS, `X-Permitted-Cross-Domain-Policies`, and `Cache-Control: no-store`.
- BUG-16: Added 10-minute frontend polling timeout and a background lifespan task to auto-flag stuck invoices.
- BUG-17: Implemented `ContentLengthLimitMiddleware` to block files >20MB before parsing begins.
- BUG-18: Hardened PIL against decompression bombs by dropping the max pixel limit to 20M.
- BUG-19: Tuned the SQLAlchemy DB connection pool configuration.
- BUG-20: Added a frontend skeleton loader for the PDF viewer.
- BUG-21: Added pyzbar startup availability checks mapped to the `/health` endpoint.
- BUG-22: Refactored `review.py` to only store JSON diffs in the audit log for EDITED actions.

Additionally, generated the Alembic migration for `users.refresh_token_hash` and `invoices.updated_at` and applied it to the database. Removed conflicting proxy settings from `vite.config.ts`.
